# Vet Shifter

Web app to help veterinary clinics find shifters, and shifters find jobs.

## Integration overview (Backend ↔ Frontend)

- **Backend** exposes HTTP endpoints (Gin) and documents them using **Swagger** generated by `swaggo/swag`.
- **Frontend** consumes the Swagger JSON and generates a typed API client using **Orval**.

The integration flow is:

1) Change backend routes / request / response types (and Swagger annotations).
2) Regenerate Swagger JSON on backend.
3) Regenerate Orval client on frontend.

## Generate Swagger on backend (Windows / PowerShell)

Swagger is generated from Go annotations (see `backend/cmd/api/main.go` and controller comments).

From the repo root:

```powershell
cd backend
go install github.com/swaggo/swag/cmd/swag@latest
swag init -g cmd/api/main.go --parseDependency -o cmd/api/docs
```

This updates:
- `backend/cmd/api/docs/swagger.json` (the file Orval reads)
- `backend/cmd/api/docs/swagger.yaml`

You can view Swagger UI when running the API at:
- `GET /swagger/index.html`

## Update Orval client on frontend (Windows / PowerShell)

Orval reads the backend Swagger JSON configured in `frontend/orval.config.ts`:
- input: `../backend/cmd/api/docs/swagger.json`
- output: `frontend/src/api/generated/api.ts`

From the repo root:

```powershell
cd frontend
npm install
npm run gen:api
```

If Orval fails because the Swagger file is missing/outdated, run the backend Swagger generation step first.
